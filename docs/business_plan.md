# Volumix ビジネスモデル設計案

個人開発のデスクトップアプリとして、管理コストを抑えつつ収益化するための設計案です。「業務効率化（時短）」に価値を置くモデルを推奨します。

## 1. 推奨ビジネスモデル：フリーミアム (Freemium)

「無料で使って良さを実感してもらい、ヘビーユースする場合に課金してもらう」モデルです。

| プラン | 価格イメージ | ターゲット |
| :--- | :--- | :--- |
| **Free (試用版)** | 無料 | 一般ユーザー、機能を試したい人 |
| **Pro (正式版)** | $10 ~ $29 (買い切り) | 動画クリエイター、配信者、編集者（業務利用） |

※ デスクトップユーティリティの場合、サーバー維持費がかからない限り「月額サブスク」より「買い切り」の方が購入ハードルが低く、好まれる傾向にあります。

---

## 2. 機能制限のアイデア (Free vs Pro)

「Free版でも機能はするが、Pro版にすると圧倒的に便利になる」という線引きが重要です。

### A案：一括処理の制限（推奨）
Volumixの最大の価値は「複数ファイルを一気に入れられること」です。ここを差別化します。

| 機能 | Free版 | Pro版 | 実装難易度 |
| :--- | :--- | :--- | :--- |
| **ファイル数** | **一度に1ファイルまで** | **無制限** | 低 |
| **動画の長さ** | 無制限 | 無制限 | 低 |
| **サンプリングレート** | 44.1kHz / 48kHz | 96kHz も可 | 低 |

> **理由**: 「1つずつなら無料でできる」という逃げ道を残すことで、ユーザーは安心して試せます。その上で「10個のファイルを1つずつやるのは面倒だ、千円なら払おう」と思わせる導線です。
> 
> **追加制限（Rate Limit）**:
> 「30分間に10ファイルまで」といった制限を併用することで、人力やスクリプトによる過度な連続処理（業務利用に近い行為）を抑制します。これは一般的なライトユーザーには影響しない緩やかな制限です。

### B案：出力の長さ制限
| 機能 | Free版 | Pro版 | 実装難易度 |
| :--- | :--- | :--- | :--- |
| **動画の長さ** | **最初の3分まで** | **無制限** | 中 |
| **透かし** | なし | なし | - |

> **理由**: 長時間の動画を扱うユーザー（配信アーカイブなど）には強力な制限ですが、Shorts動画メインのユーザーには課金動機が弱くなります。

### C案：試用回数/期間制限
| 機能 | Free版 | Pro版 | 実装難易度 |
| :--- | :--- | :--- | :--- |
| **利用回数** | **累計10回まで** | **無制限** | 中 |
| **試用期間** | 7日間 | 無制限 | 中 |

> **理由**: シンプルですが、たまにしか使わないユーザーは離脱します。また、ローカルアプリでの回数管理は回避されやすい（再インストール等）という技術的課題があります。

## 3. 販売・配布プラットフォーム

自前で決済システムを作るのは大変なので、既存のプラットフォームを利用します。

| プラットフォーム | 特徴 | 手数料 | おすすめ度 |
| :--- | :--- | :--- | :--- |
| **Lemon Squeezy** | **超おすすめ**。MoR(販売代行)モデルで税金処理もお任せ。APIが開発者向けに完全に設計されている。UIが美しい。 | 5% + 50¢ | ★★★★★ |
| **Gumroad** | 定番だが手数料が高くなった。日本円出金が可能。 | 10% (高め) | ★★★★☆ |
| **BOOTH** | 日本向けなら最強。ただしライセンスキー自動発行APIなどシステム連携は弱い。 | 低い (5.6%~) | ★★★☆☆ |
| **Stripe** | 自前実装の負担が大きい。SaaS向け。 | 安い (3.6%) | ★★☆☆☆ |

### なぜ Lemon Squeezy が最適か？
最近の開発者トレンドは圧倒的に Lemon Squeezy です。

1.  **Merchant of Record (MoR)**: 面倒な消費税(VAT)の計算・納税を彼らが代行してくれます（Stripeだと自分でやる必要がある）。
2.  **ライセンスキー機能**: 標準で付いています。ユーザーが購入するとキーが届き、アプリからそのキーを照合・アクティベーションするAPIが簡単に使えます。
3.  **手数料が安い**: Gumroad (10%) に比べて Lemon Squeezy (5% + 50¢) は利益率が良いです。
4.  **グローバル対応**: 海外販売も視野に入れるならベストな選択肢です。

### 実装イメージ (Lemon Squeezy版)
1.  Lemon Squeezyで商品登録し「License Keys」を有効にする。
2.  アプリ実装:
    ```javascript
    // 擬似コード: ライセンス認証
    async function activateLicense(licenseKey) {
      const response = await fetch('https://api.lemonsqueezy.com/v1/licenses/activate', {
        method: 'POST',
        body: JSON.stringify({ license_key: licenseKey, instance_name: 'Volumix' })
      });
      return response.json(); // { valid: true, ... }
    }
    ```

## 4. 今後のステップ案

1. **Free/Proの仕様決定**: 「A案: ファイル数制限」で進めるか決める。
2. **ライセンス認証の実装**: Gumroad APIを使った検証ロジックをアプリに組み込む。
3. **ストアページの準備**: スクリーンショットや説明文を用意する。

まずは**「A案（一度に1ファイルまで制限）」**で実装を進めてはいかがでしょうか？
UIへの変更も少なく、最も実装コストが低く、かつビジネス効果が高いと思われます。
